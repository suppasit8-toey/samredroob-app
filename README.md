# SAMREDROOB - Next.js Migration

This project has been migrated to **Next.js 15**. It is configured for deployment on **Vercel** with **Supabase** (Database) and **Cloudinary** (Media) integration.

## üöÄ Getting Started

### Local Development
1.  Install dependencies:
    ```bash
    npm install --legacy-peer-deps
    ```
2.  Run the development server:
    ```bash
    npm run dev
    ```
3.  Open [http://localhost:3000](http://localhost:3000) with your browser.

    > **Note**: If `npm run dev` fails locally, ensure you have a clean `node_modules` installation. Remove `node_modules` and `package-lock.json` before reinstalling.

## ‚òÅÔ∏è Deployment (Vercel)

The easiest way to deploy is to use the [Vercel Platform](https://vercel.com/new).

1.  Push this repository to GitHub.
2.  Import the project into Vercel.
3.  Vercel will automatically detect Next.js.
4.  **Environment Variables**: You must set these in Vercel project settings:
    - `NEXT_PUBLIC_SUPABASE_URL`: Your Supabase Project URL.
    - `NEXT_PUBLIC_SUPABASE_ANON_KEY`: Your Supabase Anon Key.
    - `NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME`: Your Cloudinary Cloud Name.
    - `NEXT_PUBLIC_CLOUDINARY_API_KEY`: Your Cloudinary API Key.

## üóÑÔ∏è Database (Supabase)

1.  Create a new project on [Supabase](https://supabase.com).
2.  Go to the **SQL Editor** and create the `products` table:
    ```sql
    create table public.products (
      id bigint generated by default as identity primary key,
      name text not null,
      category text check (category in ('curtain', 'wallpaper')),
      price text,
      image_url text,
      created_at timestamp with time zone default timezone('utc'::text, now()) not null
    );
    ```
3.  Insert sample data or use the dashboard to add products.

## üñºÔ∏è Media (Cloudinary)

1.  Create an account on [Cloudinary](https://cloudinary.com).
2.  Use the Cloudinary dashboard to upload product images.
3.  Copy the image URLs and store them in your Supabase `products` table.
